\subsection{Aufbau von TCP-Verbindungen und Durchführung von Anfragen}

Für alle Anfragen wurde ein einfaches Python Script verwendet, welches die TCP-Verbindung aufbaut, die HTTP Anfrage sendet und die Antwort empfängt~\ref{code:connection}.

\codeblock{../connection.py}{Python Script zum Aufbau von TCP-Verbindungen und Durchführung von HTTP Anfragen}{code:connection}

Das Script nutzt für die Anfragen die \texttt{httpx} Bibliothek~\cite{httpx_http2}, welche die TCP Verbindung für HTTP Anfragen vereinfacht.
Es werden die HTTP Versionen 1.1 und 2 unterstützt, welche über die Kommandozeile ausgewählt werden können~\ref{code:connection-help}.
Das Script gibt die Antwort des Servers auf der Kommandozeile aus.

\codeblock{res/connection_help.txt}{Ausgabe der Python Script Optionen über den Befehl \texttt{python connection.py --help}}{code:connection-help}

\FloatBarrier

Die HTTP-Kommunikation zwischen dem Client und dem Server wurde über Wireshark analysiert.
Dabei wurden die folgenden Schritte durchgeführt:

\begin{enumerate}\label{enum:wireshark_capture_method}
    \item Starten von Wireshark und Auswahl der entsprechenden Netzwerkschnittstelle, um den Datenverkehr zu erfassen.
    \item Anfrage an die Webseite mit dem Python Skript~\ref{code:connection} senden, um die HTTP-Kommunikation zu initiieren.
    \subitem \texttt{python3 connection.py --http\_version ...}
    \item Aufgezeichnete Pakete nach der Anfrage durchsuchen.
    \item Filterung der Frame Pakete Nummern um nur DNS Abfrage, TCP Handshake und HTTP Pakete zu betrachten.
    \subitem \texttt{frame.number >= \{ErstesPaketNummer\} \&\& frame.number <= \{LetztesPaketNummer\}}
    \item Speichern der relevanten Pakete für die weitere Analyse.
    \item Analyse der HTTP Pakete, um die Kommunikation zwischen Client und Server zu verstehen, einschließlich der Anfragen und Antworten, die über HTTP gesendet wurden.
\end{enumerate}

Zur Analyse der Kommunikation wurden einige Filterbefehle in Wireshark verwendet, um die relevanten Pakete zu identifizieren und zu isolieren.

\begin{enumerate}
    \item \texttt{frame.number >= \{ErstesPaketNummer\} \&\& frame.number <= \{LetztesPaketNummer\}}:\newline Dieser Filter wurde verwendet, um die Pakete zu isolieren, die während der HTTP-Kommunikation zwischen Client und Server ausgetauscht wurden. Dabei wurden die Paketnummern des ersten und letzten relevanten Pakets angegeben, um nur diese Pakete anzuzeigen und nur diese in die Datei zu exportieren.
    \item \texttt{http} und \texttt{http2}: Dieser Filter wurde verwendet, um nur die HTTP-Pakete anzuzeigen, die während der Kommunikation ausgetauscht wurden. Dadurch konnten die HTTP-Anfragen und -Antworten isoliert und analysiert werden, ohne von anderen Paketen wie DNS oder TCP Handshake Paketen abgelenkt zu werden.
    \item \texttt{tcp} Dieser Filter wurde verwendet, um die TCP-Pakete anzuzeigen, die während der Kommunikation ausgetauscht wurden. Dadurch konnten der Aufbau der TCP-Verbindung und die Übertragung der HTTP-Pakete über TCP analysiert werden.
    \item \texttt{http2.stream\_id == ...}: Dieser Filter wurde verwendet, um die HTTP/2-Pakete anzuzeigen, die zum einem bestimmten Stream gehören. So kann beispielsweise nur der Stream mit der Anfrage oder nur der Stream mit dem Einstellungsaustausch angezeigt werden, um die Kommunikation in diesem Stream genauer zu analysieren.
\end{enumerate}

Nach dem Filtern der Frames wurden mit Wireshark die Details der einzelnen Pakete analysiert.
Dafür wurde das Aufklappen der entsprechenden Netzwerkschicht genutzt, so sind Details wie Größe der Pakete, Flags, Header und Payload sichtbar, um die Kommunikation zwischen Client und Server zu verstehen.~\ref{fig:wireshark_capture_method}
